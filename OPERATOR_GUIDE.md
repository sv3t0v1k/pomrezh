# Руководство оператора — Run-of-Show (сценарный пульт)

Краткая инструкция по работе с интерфейсом управления сценарием мероприятия.

---

## 1. Быстрый старт

### Запуск проекта

**Вариант A — скрипт (macOS)**  
В терминале из папки проекта:

```bash
chmod +x start.sh
./start.sh
```

Скрипт проверит Node.js, зависимости, порт (по умолчанию **3000**), запустит сервер и откроет браузер. Другой порт: `PORT=3001 ./start.sh` (тот же порт должен быть у переменной окружения и в адресе браузера).

**Вариант B — вручную**

```bash
npm install
npm start
```

Сервер слушает **http://localhost:3000** (порт можно задать через `PORT`, см. `server.js`).

### Окно оператора

Главное окно приложения — это **панель оператора** (`index.html`): список событий (cue), таймер, кнопки мониторов и горячие клавиши. Блок **«Настройки и редактирование»** (шоу-файлы, OSC, импорт Excel, добавление действия/примечания) свёрнут по умолчанию; состояние открыто/закрыто запоминается в браузере. Отдельно открываются **экраны** для зала, техники и ведущего (см. раздел 5).

---

## 2. Управление сценарием

| Действие | Описание |
|----------|----------|
| **СТАРТ** | Делает **это** событие текущим в эфире: обновляются внешние экраны, запускается/сбрасывается таймер слота (если включён автостарт). |
| **ГОТОВО** | Отмечает событие выполненным (учёт «выполнено»), **не** переключает эфир на другое событие. |
| **+ Действие** | Добавить новое событие в конец списка. **Текущий cue в эфире не меняется** — правка только расширяет сценарий. |
| **Удалить** (крестик) | Удалить строку; если удаляется текущее активное — активируется следующее. |
| **Тип события** | Клик по иконке слева от номера циклически меняет тип: номер / ведущий / техника. |

### Drag & drop

Перетаскивание за область содержимого строки (**не** за кнопки) меняет порядок сценария. После перестановки состояние синхронизируется с экранами.

---

## 3. Таймер

- **Автостарт:** при нажатии **СТАРТ** на событии таймер слота запускается заново (отсчёт времени для текущего cue).
- **Пауза / Продолжить:** кнопки в блоке **ТАЙМЕР** или клавиша **P** (латинская): переключение «идёт / на паузе» без сброса накопленного времени.
- **Отображение:** в блоке **ТАЙМЕР** показывается **прошло / длительность слота** и статус (**● РАБОТА** / **❚❚ ПАУЗА**).

---

## 4. Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| **Space** | Перейти к **следующему** событию в списке (как следующий cue по сценарию), с перезапуском таймера как при **СТАРТ**. |
| **P** | Пауза / продолжить таймер. |
| **↑** / **↓** | Подсветка строки в списке **без** активации в эфире (навигация глазами). |

**Важно:** в полях ввода (текст события, время, примечания и т.д.) эти сочетания **не** обрабатываются как горячие клавиши пульта — работает обычный ввод текста.

---

## 5. Экраны мониторов

| Кнопка | Назначение |
|--------|------------|
| **Сцена** | Экран для зала: текущее, следующее, таймер. |
| **Техника** | Для техкоманды: текущее, следующее, очередь, таймер. |
| **Ведущий** | Для ведущего: текущее, следующее, заметки, таймер. |

Окна открываются через **МОНИТОРЫ**; закрытие — крестиком окна браузера.

### Индикаторы рядом с кнопками

- **●** (зелёный) — окно этого типа **открыто**.
- **○** (серый) — окно **закрыто**.

Статус обновляется при открытии/закрытии и периодически для надёжности.

---

## 6. Визуальные обозначения в списке

- **Красная рамка и красная полоса слева** — **текущее** активное в эфире событие.
- **Зелёная рамка и зелёная полоса слева** — **следующее** по сценарию событие.
- **Нейтральный** фон — остальные строки.
- **Голубая обводка** — временная подсветка при навигации **стрелками** (ещё не активировано в эфир).

## 7. Типы событий

| Тип | Иконка | Смысл |
|-----|---------|--------|
| Номер (performance) | 🎵 | Выступление / номер |
| Ведущий (host) | 🎤 | Блок ведущего |
| Техника (tech) | ⚙️ | Технический блок |

Иконки видны в списке и на выходных экранах.

## 8. Формат Excel-файла для импорта

Импорт обрабатывается в коде по **индексам колонок** (A = 0, B = 1). **Первая строка листа всегда пропускается** — считается строкой заголовков, даже если там данные.

### Структура таблицы (как в программе)

| Колонка | Поле в системе | Обязательно | Формат | Пример |
|---------|----------------|---------------|--------|--------|
| **A** | Название события | Условно | Текст | `Открытие шоу` |
| **B** | Длительность слота (время в строке действия) | Нет | См. ниже | `10` или `10:00` |

- Колонок **C, D и далее в импорте не читаются** — тип события и заметки из Excel **не подставляются**. После загрузки у всех импортированных строк тип **«Номер» (performance)**; тип можно сменить вручную (иконка слева от номера). Примечания добавляются только в интерфейсе пульта.

### Правила заполнения

**Колонка A — название**

- Берётся как текст первой ячейки строки: `row[0]`.
- Если ячейка пустая, подставляется текст **«Без названия»**.
- Строка обрабатывается, если у неё есть **хотя бы одна ячейка** (`длина строки > 0`).

**Колонка B — длительность слота**

- Берётся как строка: `row[1]` → попадает в поле времени действия и парсится как **длительность** для таймера на экранах.
- **Пустая ячейка** допустима: для слота будет использован **запасной интервал по умолчанию** (10 минут), пока не отредактируете время вручную.
- Распознавание (см. `parseDurationSecondsFromPlanned` в коде):
  - **Одно число без двоеточия** — это **минуты**: `10` → 10 минут, `5` → 5 минут.
  - **Два числа `мм:сс`** — минуты и секунды: `10:00` → 10 мин 0 сек, `2:30` → 2 мин 30 сек.
  - **Три числа `ч:мм:сс`** — часы, минуты, секунды: `1:30:00` → 1 ч 30 мин 0 сек.
- Некорректный текст может дать пустой разбор — тогда сработает тот же **fallback по умолчанию** на экранах.

**Тип и заметки**

- Из Excel **не импортируются**. Типы: **performance / host / tech** задаются только в пульте после импорта. Заметки для экрана ведущего — только через блок примечаний в интерфейсе.

### Поведение при загрузке файла

1. Читается **первый лист** книги (`.xlsx` / `.xls`).
2. **Строка 1** не импортируется (заголовок).
3. Со **строки 2** каждая непустая строка становится событием; в конец списка добавляется финальный блок **«Мероприятие окончено»** (как в интерфейсе).
4. Текущий сценарий **полностью заменяется**; состояние таймера сбрасывается; после синхронизации первым «текущим» в списке считается **первое** импортированное действие (`currentIndex = 0`).
5. Во время разбора показывается **индикатор загрузки**.

### Пример минимального файла

| | A | B |
|---|---|---|
| 1 | Название | Время (мин. или мм:сс) |
| 2 | Открытие | 5 |
| 3 | Номер артиста | 15:00 |
| 4 | Финал | 1:00:00 |

*(Строка 1 будет пропущена при импорте; в таблице она может быть любой — программа всё равно не использует её как данные.)*

### Подготовка в Excel / Google Sheets

1. В первой строке удобно указать подписи колонок (они будут пропущены).
2. Со второй строки — события: колонка **A** — название, **B** — длительность в поддерживаемом формате.
3. Сохранить как **.xlsx** или **.xls**.
4. В пульте: **Загрузить сценарий** → выбрать файл.

### Частые ошибки

| Ситуация | Пояснение |
|----------|-----------|
| Ожидали длительность в **секундах** одним числом | Сейчас **одно число без `:`** — это **минуты**, не секунды. Для «2 минуты» укажите `2` или `2:00`. |
| Нет колонки с типом / заметками | Их **нет в импорте** — только A и B. |
| «Пропала» первая строка данных | Первая строка листа **всегда** считается заголовком и не импортируется. |

## 9. Шоу-файлы (JSON) и OSC

### Шоу-файл

В блоке **ШОУ-ФАЙЛЫ** можно сохранить весь сценарий вместе с настройками OSC и состоянием таймера:

| Элемент | Назначение |
|---------|------------|
| **Название шоу** | Подпись в `meta` и имя файла при экспорте. |
| **Сохранить** | Запись в **localStorage** браузера и скачивание файла `*.show.json`. |
| **Загрузить** | Импорт JSON с диска; перед применением запрашивается **подтверждение** (несохранённые правки будут потеряны). |
| **Новое шоу** | Очистка списка событий и сброс таймера; подтверждение обязательно. |
| **Экспорт JSON** | Только скачать файл, без перезаписи localStorage (удобно для бэкапа). |
| **Список сохранённых шоу** | Быстрая загрузка из localStorage (тоже с подтверждением). |
| **Удалить** | Удалить выбранное сохранение из браузера. |
| **Автосохранение** | Через ~3 с после изменений снова вызывается сохранение в localStorage (без лишних скачиваний файла). |

Импорт **Excel** по-прежнему работает как раньше; после импорта имя шоу в поле подставляется из имени файла Excel.

### Формат файла (кратко)

В JSON есть поля `meta` (название, даты, версия), `settings.osc` (глобальные параметры OSC), массив `actions` (включая опциональный блок `osc` на каждый cue), блок `state` (индекс текущего cue, таймер и т.д.).

### OSC

Браузер **не отправляет UDP сам**; при включённой галочке **«Включить отправку»** сообщения уходят на **локальный сервер** (`npm start`), а он пересылает их по **UDP** на указанные **IP** и **порт** (например Resolume, TouchDesigner на `127.0.0.1:7000`).

- **Глобальные команды:** пауза таймера, продолжить, переход к следующему cue (**Space**), опционально **конец шоу** (`/show/end` при **ГОТОВО** на последнем в эфире событии). В полях уже подставлены типовые адреса; если поле адреса очистить, снова применятся значения по умолчанию.
- **ГОТОВО:** при отметке строки выполненной, если это **текущий в эфире** cue, отправляется per-cue **onEnd**; если это **последнее** событие в списке — дополнительно глобальная команда конца шоу, уведомление «Шоу завершено» и сброс таймера. Переключение cue кнопкой **СТАРТ** / Space по-прежнему шлёт onEnd предыдущего cue отдельно (дубля с **ГОТОВО** нет, если не жать оба подряд на одном переходе).
- **По каждому событию:** в строке раскройте **OSC команды** — **при старте** и **при завершении**. По умолчанию: `/cue/start` с аргументом `{title}` и `/cue/end` с `{id}`; очистка адреса снова включает эти дефолты.

Если OSC **выключен** галочкой «Включить отправку», UDP не уходит. Пустой адрес в интерфейсе не отключает глобальные команды навсегда — подставляются значения по умолчанию (см. подписи в блоке OSC).

### Отладка OSC

| Где | Что сделать |
|-----|-------------|
| **Сервер** | Запуск с логами: `OSC_DEBUG=1 npm start` — в терминале видно тело запроса, цель UDP, размер пакета, успех или ошибку отправки. |
| **Браузер** | В консоли (F12) при загрузке — краткая сводка; подробные логи `sendOSC`: `localStorage.setItem('OSC_DEBUG','1')` и обновить страницу, либо открыть пульт с параметром `?oscdebug=1`. |
| **Тест** | Кнопка **Тест OSC** шлёт `/ping` на выбранный IP:порт; приёмник для проверки: `npm run test:osc-receiver` (порт **7000**, не должен быть занят другой программой). |

## 10. Решение проблем

| Проблема | Что проверить |
|----------|----------------|
| Экран монитора не открывается | Блокировка **всплывающих окон** в браузере; разрешить для `localhost`. |
| Таймер или экраны «отстают» | Обновить страницу пульта (**F5**); закрыть лишние окна экранов и открыть снова. |
| Не перетаскиваются строки | Тянуть за область **содержимого** строки; не зажимать **Ctrl** для обычного drag в браузере. |
| Нет звука / не то окно | Убедиться, что открыт именно адрес с пультом: **http://localhost:3000**. |

### Добавление строки «+ Действие»

Новая строка **не становится** активным cue: в эфире остаётся то же событие, что и до добавления (пока вы сами не нажмёте **СТАРТ** на другой строке).
