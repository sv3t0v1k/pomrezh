## 3. Управление сценарием (cue)

| № | Проверка | Ожидаемое поведение | Реальность | Статус |
|---|----------|---------------------|------------|--------|
| 3.1 | Добавить действие (`Правка -> Добавить действие`) | В списке появляется новое действие, state обновляется. |  |  |
| 3.2 | Добавить заметку (`Правка -> Добавить заметку`) | Создается заметка и корректно привязывается к действию. |  |  |
| 3.3 | Кнопка `▶` (GO) | Cue становится текущим (`active`), UI обновляется сразу. |  |  |
| 3.4 | Кнопка `✓` (DONE) на текущем cue | Cue отмечается completed, отправляется onEnd, текущий cue не переключается автоматически. |  |  |
| 3.5 | `✓` на неактивном cue | Меняется только completed-состояние, текущий cue не сбивается. |  |  |
| 3.6 | Drag-and-drop reorder | Порядок меняется без рассинхрона, `appState.actions` совпадает с DOM. |  |  |
| 3.7 | Удаление cue | Cue удаляется корректно, нумерация обновляется, state синхронизируется. |  |  |
